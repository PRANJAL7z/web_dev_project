WORKFLOW EXAMPLE: REGISTERING A CRIME
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: CITIZEN REPORTS CRIME
↓
Officer opens CrimeSense system in browser

STEP 2: OFFICER ENTERS DATA
↓
- Clicks "Register New FIR"
- Fills form:
  * Complainant name: "John Doe"
  * Crime type: "Theft"
  * Description: "Mobile phone stolen at market"
↓
Clicks "Register FIR"

STEP 3: FRONTEND SENDS DATA
↓
JavaScript collects form data
Sends POST request to backend: /api/firs
Data: { name: "John Doe", type: "Theft", description: "..." }

STEP 4: BACKEND PROCESSES
↓
Flask receives request
Generates unique ID: FIR101
Calls AI Sentiment Analyzer

STEP 5: AI ANALYZES URGENCY
↓
Sentiment Analyzer reads description
Finds keywords: "stolen" (medium priority)
Calculates importance score: 0.55
Returns score to backend

STEP 6: DATABASE SAVES
↓
Backend inserts into database:
INSERT INTO firs VALUES (
  'FIR101', 'John Doe', 'Theft', '13/11/2025',
  'Registered', 'Mobile phone stolen...', 0.55
)
Database commits (saves permanently)

STEP 7: BACKEND RESPONDS
↓
Returns JSON to frontend:
{ success: true, id: 'FIR101', fir: {...} }

STEP 8: FRONTEND UPDATES
↓
Shows success message: "FIR FIR101 registered!"
Automatically refreshes FIR list
New FIR appears in table
Officer can immediately search for it

STEP 9: CRIME ANALYSIS UPDATES
↓
Next time officer views Crime Hotspot:
- New FIR included in analysis
- Location extracted from description
- Added to city crime statistics
- Regression model recalculated
- Charts updated with new data

THE AI/ML INTELLIGENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. SENTIMENT ANALYZER (FIR Priority)
   
   What it does:
   Reads FIR description and determines urgency
   
   How:
   - Scans for urgent keywords: "critical", "emergency"
   - Scans for violence keywords: "assault", "weapon"
   - Scans for routine keywords: "minor", "routine"
   - Calculates score: 0.1 (low) to 0.9 (high)
   
   Example:
   Description: "Urgent! Armed robbery with injuries"
   Keywords found: "urgent" (+0.15), "armed" (+0.12), "injuries" (+0.12)
   Score: 0.69 (HIGH PRIORITY)
   
   Result: This FIR appears at top of list

2. CRIME PATTERN ANALYZER (Hotspot Detection)
   
   What it does:
   Finds patterns in crime data to predict future crimes
   
   How:
   - Extracts locations from all FIRs
   - Groups crimes by city
   - Calculates average severity per city
   - Uses Linear Regression: y = mx + b
     * x = number of crimes
     * y = average severity
     * Finds relationship between quantity and severity
   
   Example:
   City A: 10 crimes, avg score 0.6 → High risk
   City B: 3 crimes, avg score 0.3 → Low risk
   Regression line shows: More crimes = Higher severity
   
   Result: Predicts City A needs more officers

3. TREND PREDICTOR (Future Forecasting)
   
   What it does:
   Predicts crime trends for next 3 months
   
   How:
   Formula: predicted = current × (1 + current × 0.05 × month)
   
   Example:
   Current score: 0.6
   Month 1: 0.6 × (1 + 0.6 × 0.05 × 1) = 0.618
   Month 2: 0.6 × (1 + 0.6 × 0.05 × 2) = 0.636
   Month 3: 0.6 × (1 + 0.6 × 0.05 × 3) = 0.654
   Trend: INCREASING
   
   Result: Alert! This area needs attention

4. OFFICER DISTRIBUTOR (Resource Optimization)
   
   What it does:
   Calculates optimal officer deployment
   
   How:
   Formula: officers = (city_score / total_score) × total_officers
   
   Example:
   Total officers: 15
   City A score: 5.5 (out of total 20)
   City A officers: (5.5 / 20) × 15 = 4 officers
   
   Result: Deploy 4 officers to City A

VISUALIZATION SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. REGRESSION CHART (Pattern Discovery)
   Shows: Crime count vs Crime severity
   Purpose: Find if more crimes = worse crimes
   
   Visual:
   - Each city = colored dot
   - Green dot = Safe city
   - Yellow dot = Moderate risk
   - Red dot = Dangerous city
   - Black line = Prediction trend
   
   Insight: "Cities with more crimes tend to have worse crimes"

2. BAR GRAPH (City Rankings)
   Shows: All cities ranked by crime score
   Purpose: Quick comparison of danger levels
   
   Visual:
   - Tall red bars = Dangerous cities
   - Short blue bars = Safe cities
   - Easy to spot worst areas
   
   Insight: "These 3 cities need immediate attention"

3. TREND CHART (Future Prediction)
   Shows: Crime trends for next 3 months
   Purpose: Predict future hotspots
   
   Visual:
   - Lines going up = Crime increasing
   - Flat lines = Crime stable
   - Color per city
   
   Insight: "City X will become dangerous in 2 months"

4. OFFICER DISTRIBUTION (Resource Map)
   Shows: How many officers per city
   Purpose: Optimize police deployment
   
   Visual:
   - Cards showing officer count
   - Crime stats per city
   - Easy to see allocation
   
   Insight: "Deploy 5 officers to City A, 2 to City B"
